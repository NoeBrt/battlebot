# BATTLEBOT - AI CONTEXT & ARCHITECTURE

## 1. Project Overview
This repository holds a Java-based Battlebot simulator (Simovies) and a Python-based Reinforcement Learning (RL) framework using the CMA-ES algorithm. The goal is to optimize the continuous numerical parameters of a generic bot brain (Team A) to maximize its win rate and overall fitness against a continuous gauntlet of diverse opponent bots (Team B).

## 2. Combat Configuration & Game Rules
Each simulation match pits **Team A** against **Team B**.
- **Team Composition (Per Team)**:
  - **1 Main Bot**: `300 HP`, Movement Speed `1`, Detect Range `300`, Body Radius `50`
  - **2 Secondary Bots**: `100 HP`, Movement Speed `3`, Detect Range `500`, Body Radius `50`
- **Map Dimensions**: `3000 x 2000` units.
- **Combat Mechanics**:
  - Bullets: Damage = `10`, Velocity = `10`, Radius = `5`, Range = `1000`, Firing Latency = `20` ticks.
  - Sensors: Bots use `IRadarResult` and `IFrontSensorResult` to detect `OpponentMainBot`, `OpponentSecondaryBot`, `Wreck`, `WALL`, `BULLET`.

## 3. RL Training Loop & CMA-ES Pipeline
The AI training uses Python to orchestrate fast headless Java matches using threading.

- **Trainer Script**: `train_rl_genetic_tournament.py` (and related variants).
- **Opponent Gauntlet**: The bot evaluates its fitness against: `MacDuo`, `Himself` (mirror match), `Stage8`, `MecaMouse`, `Superhero`, `FifthElem`, `Claude`, `AdaptiveK`, `Team1`.
- **Execution Pipeline**:
  1. Python CMA-ES generates a set of mutated float parameters.
  2. Parameters are written directly into `src/algorithms/rl/RLConfig_genXXXX.java`.
  3. Python dynamically modifies `src/characteristics/Parameters.java` to set the new target opponent's class name (e.g., `teamBMainBotBrainClassName`).
  4. **CRITICAL ARCHITECTURE NOTE (Java Inlining)**: Because `Parameters.java` defines class names as `public static final String`, standard `javac` natively inlines these constants into the other compiled class files. To bypass this, the Python pipeline wipes the `beans/` binary cache and forces a full clean recompilation before fighting a new opponent type.
  5. `src/supportGUI/HeadlessMatchRunner.java` initiates the match silently, calculates metrics (Kill weights, Survival weights), and outputs `ScoreA`, `ScoreB`, `WinA` to the terminal.
  6. Python parses `stdout` to calculate the final Fitness.

## 4. Key Files & Directories Map
- `src/characteristics/Parameters.java`: The absolute source of truth for match initialization (spawn coordinates, team names, brain classes). Note the recompilation rule above when editing.
- `src/supportGUI/HeadlessMatchRunner.java`: The headless execution engine. Holds the mathematical logic defining how much "Fitness" a kill or a survival is worth.
- `src/algorithms/rl/RLConfig_best_A.java`: The template file receiving injected float parameters generated by CMA-ES.
- `tournament_live.log`: Streaming output showing detailed fitness breakdown per opponent per generation.
- `rl_results/`: Directory storing generation `.json` checkpoints and `.java` configurations.
